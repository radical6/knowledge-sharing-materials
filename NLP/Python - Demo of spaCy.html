Tokenization, part-of-speech tagging, dependency parsing¶

Tokenization is a fundamental task in NLP that involves splitting a document into smaller units called tokens.

Part-of-speech tagging is a process of marking up a token in a text as corresponding to a particular part of speech, based on both its definition and its context.

Dependency parsing is the task of analyzing the grammatical structure of a sentence and establishing the relationships between head (parent) words and the words which modify those heads, called dependents (children).

The difficulty in tokenization lies on how to get the ideal split so that all the tokens in the text have the correct meaning, and there are no left out tokens.

Let's define a function to retokenize the document based on recognized entities and noun chunks.

In [13]:
def nlp_retokenized(txt):
    doc = nlp(txt)
    orig_ents = list(doc.ents)
    fixed_ents = []
    # Ensure inclusion of currency symbol in entities
    for ent in orig_ents:
        if ent.n_lefts > 0:
            # If there are left-children, check if one of them is a symbol and nmod
            currency_symbol = [tok for tok in ent.lefts if tok.dep_ == "nmod" and tok.pos_ == "SYM"]
            if currency_symbol:
                fixed_ents.append(doc[currency_symbol[0].i:ent.end])
            else:
                fixed_ents.append(ent)
        else:
            fixed_ents.append(ent)
    # Merge entities and noun chunks into one token
    spans = fixed_ents + list(doc.noun_chunks)
    # Filter a sequence of spans so they don't contain overlaps
    spans = spacy.util.filter_spans(spans)
    with doc.retokenize() as retokenizer:
        for span in spans:
            retokenizer.merge(span)
    return doc

def extract_currency_relations(doc):
    relations = []
    for money in filter(lambda w: w.ent_type_ == "MONEY", doc):
        if money.dep_ in ("attr", "dobj"):
            # attr - attribute, dobj - direct object, nsubj - nominal subject
            subject = [w for w in money.head.lefts if w.dep_ == "nsubj"]
            if subject:
                subject = subject[0]
                relations.append((subject, money))
        elif money.dep_ == "pobj" and money.head.dep_ == "prep":
            # pobj - object of preposition, prep - prepositional modifier
            relations.append((money.head.head, money))
    return relations

Word embedding¶

Word embedding is the collective name for a set of language modeling and feature learning techniques in NLP where words are mapped to vectors of real numbers, aptly called word vectors. Word vectors represent words as multidimensional continuous floating-point numbers where semantically similar words are mapped to proximate points in geometric space.

spaCy's Large English model come with GloVe (an unsupervised learning algorithm for obtaining vector representations for words) vectors trained on Common Crawl (datasets obtained from crawling the web).

Cosine similarity¶

Cosine similarity is a metric used to determine how similar the two vectors are by calculating the cosine of the angle between two vectors. The cosine function gives 1 at θ = 0° and -1 at θ = 180°, which means that for two overlapping vectors, cosine will be highest, while cosine will be lowest for opposite vectors. The closer the two vectors are by angle (the smaller the angle), the higher the cosine similarity.

For word embeddings, if two words frequently appear together in the same context, then their corresponding word vectors will be similar. Secondly, if there is no relation between the contexts in which each of the two words appear, then their word vectors will be orthogonal. Lastly, if there is a notable difference between the contexts in which each of the two words appear, then their word vectors will be opposite.

In [20]:
nlp("staff").similarity(nlp("employees"))
Out[20]:
0.6675002052314796

Two words being antonyms does not imply their word vectors are opposite. In fact, word vectors of antonyms will generally be similar as they frequently appear together in the same context.
